# ZStack 快速入门

## 架构概览

ZStack 是一款开源的 IaaS 云管理软件，采用全异步架构设计，核心组件包括：

- **管理节点（Management Node）**：控制平面入口，处理所有 API 请求
- **计算节点（Compute Node）**：运行虚拟机的物理服务器
- **存储**：主存储（运行 VM 使用）和备份存储（镜像和快照存放）
- **网络**：L2/L3 网络模型，支持扁平网络、VPC、安全组等

## 资源层级关系

```
区域（Zone）
 └── 集群（Cluster）
      └── 物理机（Host）
           └── 云主机（VM Instance）
                ├── 系统盘（Root Volume）
                ├── 数据盘（Data Volume）
                └── 网卡（VM NIC）
```

## 常见运维操作

### 创建云主机

创建 VM 需要以下前置资源：
1. 计算规格（Instance Offering）— 定义 CPU、内存
2. 镜像（Image）— 操作系统模板
3. L3 网络（L3 Network）— 网络连接

### 物理机维护

进入维护模式前需要注意：
- 确认其他物理机有足够容量承接 VM 迁移
- 运行中的 VM 会自动热迁移
- 维护模式下无法创建新 VM

### 存储扩容

主存储使用率超过 80% 时建议扩容：
- Ceph 存储：添加 OSD 节点
- 本地存储：扩展磁盘或添加新物理机
- NFS/SAN：扩展后端存储容量

## 故障排查指南

### VM 无法启动

1. 检查物理机状态：是否 Connected
2. 检查镜像状态：是否 Ready
3. 检查存储空间：主存储是否有足够空间
4. 检查网络：L3 网络是否有可用 IP

### 网络不通

排查步骤：
1. 检查 VM 网卡是否分配到 IP
2. 检查安全组规则是否放行
3. 检查 VPC 路由器状态
4. 检查 EIP 绑定状态（公网访问场景）

### 存储告警

常见原因：
- 快照过多占用空间 → 清理过期快照
- 大数据盘未释放 → 检查已删除 VM 的残留云盘
- 镜像重复 → 清理不需要的镜像

## 最佳实践

### 容量规划

- CPU 超分比建议不超过 4:1
- 内存超分比建议不超过 1.5:1
- 存储预留 20% 空间用于快照和临时数据
- 单集群物理机数量建议不超过 100 台

### 备份策略

- 关键业务 VM：每日快照，保留 7 天
- 测试环境：每周快照，保留 3 天
- 数据库类 VM：结合应用层备份 + 云盘快照双重保障

### 安全加固

- 使用 IAM2 进行细粒度权限控制
- 不同业务使用不同 VPC 网络隔离
- 定期审计安全组规则
- 启用 ZWatch 告警监控异常行为
